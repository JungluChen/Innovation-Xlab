    Credibility section

    Every interface change was traced to observed user friction.

    The design sprint used workflow mapping, unguided testing, and friction coding to identify where users lost trust, selected the wrong path, or could not recover from unclear system states.

    01

    Workflow Mapping

    Map document logic before interface fixes.

    • Map document types across AP and BOM reconciliation flows.
    • Identify matching keys and trace variance decisions.
    • Capture exception paths where extraction alone fails.
    • Translate workflow ambiguity into product requirements.
    02

    Task-Based Testing

    Observe users without steering their choices.

    • Run more than 15 unguided think-aloud sessions.
    • Record hesitation, misclicks, confusion, and abandonment.
    • Compare behavior across 11 participant cohorts.
    03

    Success Framework

    Evidence-based coding prioritized repeated friction.

    • Apply the 90 percent No-Help Rule while keeping researchers passive.
    • Use evidence-based coding rather than subjective visual preference.
    • Prioritize repeated friction logs into concrete product fixes.
    • Keep core evidence visible without hover-only disclosure.

    Proof-of-evolution section

    Research findings became concrete workflow fixes.

    Each change below connects an observed user problem to a specific product response and a practical expected impact on trust, recovery, and workflow clarity.

    Problem

    Users could not connect what they asked Fibula to do with what the system was building. Silent processing made the workflow feel disconnected from the user's mental model.

    Fix

    Add live workflow canvas animation that draws nodes as the system synthesizes them.

    Expected impact

    Users can see progress, understand system activity, and trust that the workflow is being constructed from their intent.

    Problem

    Verbose option lists and too many visible choices caused users to pick the quickest path rather than the correct path.

    Fix

    Reduce options into predictive chips, progressive disclosure, and fluid expanding tables.

    Expected impact

    Users make faster choices with less cognitive burden while still preserving access to deeper configuration when needed.

    Problem

    Silent errors and dead-end modals left users without a recovery path.

    Fix

    Add natural-language advice cards, in-modal rerun shortcuts, and clearer next-step guidance.

    Expected impact

    Users can recover from failure states without leaving the workflow or waiting for technical support.

    Strategic analysis section

    The redesign moves automation from hidden execution to visible trust.

    Fibula reduces configuration ambiguity, increases exception visibility, and makes workflow state explicit so users can validate automation without losing decision control.

    Panel 1 | Mental Model Gap

    Users needed visible system guidance, not silent synthesis.

    • Users expected immediate guidance from the first action.
    • Live feedback was needed while workflows were being synthesized.
    • The interface had to show how natural-language intent became executable structure.
    • Better state visibility reduced uncertainty and trust gaps.

    Panel 2 | Reconciliation Risk Mitigation

    Clear exception visibility protects industrial decisions.

    • Reconciliation risk comes from missed variance, unclear output, and untraceable decisions.
    • Fibula should turn outputs into clear decision matrices.
    • Exception states should be visible and recoverable.
    • Human validation remains part of the automation loop.
